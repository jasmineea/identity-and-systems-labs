# Access Review Summary

## Overview
This document summarizes the access review scenario used in the Identity Governance, Access Reviews, and JML Lab.

The goal is to demonstrate how periodic access validation can help organizations confirm that users still need assigned access.

---

## Review Target
**Example Target:** Contractors group  


---

## Review Purpose
This review was designed to validate whether current members still require access based on role, employment status, or business need.

This type of review supports:
- least privilege
- stale access reduction
- audit readiness
- controlled access retention

---

## Review Design

### Review Scope
- one governance-relevant group
- small enough to review clearly
- meaningful enough to demonstrate access recertification value

### Reviewer Model
Possible reviewer approaches:
- group owner
- manager
- administrator
- self-review *(less ideal for higher-risk access)*

### Review Frequency
Example frequencies:
- monthly
- quarterly
- semi-annually

### Review Outcomes
Typical outcomes may include:
- approve continued access
- deny continued access
- remove access if no decision is made
- escalate for manual follow-up

---

## Expected Result
Users who no longer need the selected access should be identified and removed, while legitimate users retain access.

---

## Governance Value
Access reviews help organizations:
- identify stale memberships
- confirm role-appropriate access
- reduce standing access risk
- create evidence of periodic governance activity

---

## Key Takeaways
- access should not remain indefinitely without review
- recurring review helps align identity with current business need
- access reviews are both a security and governance control
