# Glossary

## Active Directory (AD)
Microsoft’s traditional on-premises directory service used to manage users, groups, computers, and policies in Windows environments.

## Administrative Role
A built-in or custom role that grants elevated permissions to perform management tasks in a system.

## Conditional Access
A Microsoft Entra policy framework that evaluates signals such as user, device, location, or application and then applies access controls like requiring MFA or blocking access.

## Compliance Policy
In Intune, a policy that defines the rules a device must meet to be considered compliant.

## Configuration Profile
In Intune, a collection of settings that configures devices according to organizational requirements.

## DHCP
Dynamic Host Configuration Protocol. A network service that automatically assigns IP addresses and related network settings to devices.

## DNS
Domain Name System. A service that translates names like `example.com` into IP addresses that computers use to locate systems.

## Entra ID
Microsoft’s cloud identity and access management platform, formerly known as Azure Active Directory.

## Group Policy (GPO)
A Windows feature used in Active Directory environments to centrally manage and enforce settings for users and computers.

## Hybrid Identity
An identity model where an organization uses both on-premises Active Directory and cloud-based identity services such as Microsoft Entra.

## Intune
Microsoft’s cloud-based endpoint management platform used to manage devices, compliance, configuration, and application policies.

## Least Privilege
A security principle that gives users only the minimum level of access needed to perform their job.

## MFA
Multi-factor authentication. A sign-in control that requires more than one form of verification, such as a password plus a mobile prompt or code.

## Offboarding
The process of removing or disabling a user’s access when they leave an organization or no longer need access.

## Organizational Unit (OU)
A container in Active Directory used to organize users, groups, and computers for management and policy application.

## PowerShell
A task automation and configuration management tool used heavily in Windows and Microsoft environments.

## Remediation
The process of correcting a problem, such as bringing a device back into compliance or fixing a misconfiguration.

## RBAC
Role-based access control. A way of assigning permissions based on job roles rather than directly assigning access one user at a time.

## Rollback
A planned method for undoing a change if something goes wrong during implementation.

## SSO
Single sign-on. A method that allows users to access an application using their existing identity provider credentials without signing in separately.

## User Lifecycle Management
The process of creating, updating, maintaining, and removing user access throughout a person’s time with an organization.

## Validation
The process of confirming that a change or configuration worked as intended after implementation.
