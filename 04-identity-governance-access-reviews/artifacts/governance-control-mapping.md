# Governance Control Mapping

## Overview
This document maps the main lab activities to governance and control outcomes.

The purpose is to show how identity administration decisions can support broader security, audit, and compliance goals.

---

## Control Mapping Summary

| Governance Activity | Risk Addressed | Control Outcome |
|---|---|---|
| Group-based access assignment | Excessive direct access and inconsistent provisioning | Standardized and reviewable access structure |
| Access review design | Stale or unnecessary permissions | Periodic revalidation of access need |
| JML workflow documentation | Delayed access changes during hire, role change, or exit | Repeatable lifecycle-based access management |
| Contractors group review scope | Stale group membership and unnecessary retained access | Easier auditability and clearer governance boundary |
| Reviewer-defined access decisions | Informal or undocumented access retention | Clear accountability for access decisions |

---

## Governance Themes

### Least Privilege
Access should align to current business need and be removed when no longer required.

### Reviewability
Access should be easy to inspect, validate, and certify.

### Repeatability
The same lifecycle event should produce the same access outcome every time.

### Audit Readiness
Access decisions should be understandable and support evidence collection.

---

## Key Takeaways
- identity controls are stronger when they are structured and reviewable
- governance improves when access is granted through groups instead of individuals
- lifecycle documentation reduces ambiguity and access drift
- access reviews support accountability and privilege reduction
