# Validation Checklist

## Overview
This checklist is used to confirm that the identity governance change was prepared and validated correctly.

---

## Validation Items

- [ ] Contractors group identified correctly
- [ ] Current members reviewed before change
- [ ] Membership export completed
- [ ] Change objective documented
- [ ] Risk documented
- [ ] Rollout steps documented
- [ ] Rollback steps documented
- [ ] Review/cleanup target confirmed
- [ ] Post-change membership verified
- [ ] Final notes documented

---

## Validation Goal
Ensure the change is supported by evidence, documentation, and a clear recovery path.
